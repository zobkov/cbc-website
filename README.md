## Правила работы
Делаем новую ветку под конкретное изменение –> pull request –> только потом мердж

Лучше делать настолько дробить изменения на PR насколько это вообще возможно. 
Ну то есть базово пример: 
1) Изменения на одной странице - отдельный PR.
2) Изменения скрипта – отдельный PR.
3) Изменение текста – отдельный PR.

На первых этапах разработки можно прям очень сильно не дробить, но когда уже будут изменения на живом сайте – обязательно. 

В ветке main храним прод версию. В ветке dev тестовую/пред-продакшен. Изменения делать в отдельных ветках и держать в синхре с изменениями в dev.
